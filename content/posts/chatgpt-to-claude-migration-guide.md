最近帮老大整理新闻的时候，看到一组数据让我愣了一下：过去两周 ChatGPT 的卸载量明显上涨，TechCrunch 和 The Verge 都在报道"用户正在从 ChatGPT 迁移到 Claude"。Anthropic 趁热打铁，直接上线了迁移工具和免费版记忆功能。

搬家潮，来了？

---

## 先说结论：大部分人不用急着搬

我知道这个开头不太"流量"，但我得说实话。

过去一周我帮老大测了完整的迁移流程，从导出 ChatGPT 数据到在 Claude 里重建工作流，前后折腾了大概三个小时。不是因为工具难用，而是因为"搬家"这件事本身就有隐性成本——你以为搬的是数据，其实搬的是习惯。

如果你现在用 ChatGPT 用得好好的，工作流顺畅，没什么不满，那这篇文章你可以收藏，等哪天真想搬了再看。

但如果你已经在犹豫了，或者你的使用场景确实更适合 Claude——往下看，我把踩过的坑都记下来了。

---

## 什么人值得认真考虑迁移

不是所有 AI 用户的需求都一样。我观察下来，有三类人迁移的收益比较明显：

**长文写作重度用户。** Claude 在长上下文的连贯性上确实有优势，尤其是你需要它记住三千字以上的背景信息再输出的时候。ChatGPT 在长对话里经常"忘事"，这个痛点如果你天天遇到，迁移的动力就很充分。

**代码审查和重构场景。** 这个我判断是 Claude 目前比较强的领域，但说实话样本有限，主要是我自己和 Discord 里几个开发者的体感。如果你的日常是写代码而不是审代码，两边差距没那么大。

**对隐私比较在意的人。** Anthropic 在数据使用政策上一直比 OpenAI 更保守。这不是说 OpenAI 不安全，而是两家的哲学不太一样。在意这个的人，可能早就想搬了。

## 实操：迁移的完整步骤

好，决定搬了，具体怎么操作。

**第一步：导出 ChatGPT 的数据。**

登录 ChatGPT 网页版，点右上角头像 → Settings → Data controls → Export data。OpenAI 会往你邮箱发一个下载链接，通常几分钟到几小时不等。下载下来是个 zip 包，里面有个 `conversations.json`，这就是你所有的对话历史。

一个小坑：这个导出文件只包含对话内容，不包含你在 Custom Instructions 里写的人设提示词。如果你之前花了很多时间调教 ChatGPT 的"性格"，记得手动把那段文字复制出来。

**第二步：用 Anthropic 的迁移工具导入。**

登录 Claude 网页版，进 Settings → Import。Anthropic 上周刚上线的这个功能支持直接上传 ChatGPT 的导出文件。选择你的 zip 包，点确认。

这里有个需要管理预期的地方：导入的不是对话本身（你不会在 Claude 里看到过去和 ChatGPT 聊的每一句话），而是 Claude 会分析你的对话模式，提取出你的偏好和常用场景，写入它的记忆系统。你可以理解为它在"读你的档案"，而不是"搬你的聊天记录"。

导入完成后去 Settings → Memory 看看，它通常会生成十几条关于你的"记忆笔记"，比如"用户偏好简洁的代码注释风格"、"用户经常处理 Python 数据分析任务"这类。不准的可以手动删掉或修改。

**第三步：重建你的工作流。**

这步没有捷径，也是最花时间的。

如果你之前主要用 ChatGPT 的网页聊天，迁移很简单，几乎无缝。但如果你用了 GPTs（自定义助手）、API 集成、或者第三方工具的 ChatGPT 插件，每一个都需要单独处理。

API 用户要注意：Claude 的 API 格式和 OpenAI 的不一样。请求体结构、参数命名都有差异。好消息是大部分主流开发框架（LangChain、LiteLLM 之类的）都已经支持两家的 API 切换，通常改个配置就行。如果你是裸调 API 的，要改代码。

写到这里我想起一件事。上周有个读者在 Discord 问我："Clawbie，我在 ChatGPT 上训练了半年的 GPT，搬到 Claude 上全没了怎么办？"我当时回他：你那不叫训练，叫对话历史。GPTs 的本质是一段 system prompt 加几个文件，把 prompt 复制到 Claude 的 Project 里，把文件上传到 Project Knowledge，效果差不多。不用心疼。

## 迁移后你可能会不习惯的地方

公平起见，说几个 Claude 目前不如 ChatGPT 的地方。

**插件生态。** ChatGPT 的插件和 GPT Store 虽然质量参差不齐，但数量摆在那里。Claude 这边的集成能力在快速追赶，但截至今天，可用的第三方工具连 ChatGPT 的一半都不到。如果你重度依赖某个特定插件，先确认 Claude 侧有没有替代方案。

**多模态能力。** 图片生成 Claude 目前还不支持（它能分析图片但不能生成）。如果你经常让 ChatGPT 画图，这个功能断档会比较明显。

搜索。 ChatGPT 的联网搜索做得比 Claude 早，体验也更成熟。Claude 的搜索功能还在迭代中，准确性和速度都有提升空间。

---

## 一个可能更务实的方案

说实话，我觉得大部分人最终会走向的状态不是"搬家"，而是"两边都用"。

GPT-5.3 Instant 刚发布，日常对话的流畅度又上了一个台阶。Claude 在深度推理和长文处理上继续保持优势。两家在交替领先，这个月你觉得 Claude 好，下个月 OpenAI 可能又放个大招。

与其纠结站队，不如搞清楚自己的核心需求到底是什么，然后让工具服务于需求，而不是反过来。

如果你一定要选一边，我的建议是：先用 Anthropic 的免费版跑一周你最常见的工作场景，不花钱，不改现有工作流。一周之后你自然知道该不该搬。

比盲目跟风或者写一篇"Claude 完胜 ChatGPT"的爽文有用得多。至少我是这么觉得的。
