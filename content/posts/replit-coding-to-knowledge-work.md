Replit 半年内估值从 30 亿涨到 90 亿美元，翻了三倍。

我点开他们的新产品发布视频，8 分钟看完，第一反应是——这还是那个「在线写代码」的 Replit 吗？Canvas、Apps、Sites、Slides、Videos，除了代码编辑器，他们几乎把整个办公套件都做了一遍。CEO Amjad Masad 在视频里说得很直白：「我们不只是帮你写代码，我们要帮你完成所有知识工作。」

这不是 Replit 一家的转向。过去两个月，Pi 变成了 OpenClaw，Claude Code 推出了 Cowork 模式，Notion 发布了 Custom Agents。**所有做 coding agent 的公司，都在同时做一件事——往上走。**

## 代码写完了，然后呢？

两年前的 Replit 还在强调「AI 辅助编程」。那时候的主流观点是：AI 能帮你补全代码、生成函数、解释报错，但核心逻辑还得人来写。


![AI编程能力演进：从辅助到接管](/images/posts/replit-coding-to-knowledge-work.svg)
现在这些话听起来像上个时代的遗物。

2025 年 12 月之后，Claude 3.7 Opus、GPT-5.4、Gemini 2.5 这一批模型出来，编程能力直接跨过了一个台阶。不是「辅助」了，是「接管」了。你给它一个需求，它能从架构设计、代码实现、测试用例到部署脚本，全链路搞定。

Replit 的数据很能说明问题：他们平台上 AI 生成的代码占比，从 2023 年的 20% 涨到现在的 80%。用户不再是「写代码时偶尔问问 AI」，而是「全程让 AI 写，自己只负责验收」。

当编程这件事被 AI 基本解决，一个做编程工具的公司能往哪走？答案是：**往知识工作的其他环节走。**

## 所有 coding agent 都在做同一件事

Replit 的转型不是孤例。过去三个月，几乎所有头部 coding agent 公司都在扩展边界：


![Coding Agent 产品演进对比图](/images/posts/replit-coding-to-knowledge-work-2.svg)
| 公司 | 原来做什么 | 现在在做什么 |
|------|-----------|-------------|
| Replit | 在线代码编辑器 + AI 补全 | Canvas、Apps、Sites、Slides、Videos 全套办公工具 |
| Pi (→ OpenClaw) | 对话式 AI 助手 | 多模态知识工作平台，支持文档、表格、演示 |
| Claude Code | 代码生成和调试 | Cowork 模式，处理邮件、会议纪要、项目管理 |
| Cursor | AI 代码编辑器 | 正在测试文档协作和项目管理功能 |

这不是巧合。当 AI 把「写代码」这个最难的知识工作环节攻克之后，剩下的事情——写文档、做 PPT、整理数据、回邮件——在技术上反而更简单。

用 Replit 自己的话说：「如果 AI 能帮你从零写出一个完整的 Web 应用，为什么不能帮你从零做出一份演示文稿？」

## 为什么是「往上走」而不是「做深」？

有人可能会问：既然编程还没完全解决（比如复杂系统的架构设计、性能优化），为什么不继续在代码领域做深，而是急着扩展到其他领域？


![编程工具向知识工作平台扩展的三大原因](/images/posts/replit-coding-to-knowledge-work-3.svg)
答案藏在商业逻辑里。

**第一，编程市场的天花板比想象中低。** 全球专业程序员大约 2700 万人。听起来不少，但对比一下：全球知识工作者超过 10 亿人。如果你只做编程工具，你的潜在用户是 2700 万；如果你做知识工作平台，你的潜在用户是 10 亿。市场规模差了 40 倍。

**第二，用户已经在用同一个 AI 做所有事情了。** 我自己就是例子。我用 Claude 写代码、写文章、整理素材、生成图表、回复邮件。我不会因为「这是写代码的 AI」就只在写代码时用它。用户的真实需求是「一个能帮我干活的 AI」，不是「一个只会写代码的 AI」。

Replit 看到了这个趋势。与其让用户在 Replit 写完代码，再切到 Notion 写文档、切到 Canva 做设计，不如把这些功能都集成进来。用户留在你的平台时间越长，你的价值就越大。

**第三，技术栈是通用的。** 做 coding agent 和做 knowledge work agent，底层能力是一样的：理解自然语言、生成结构化内容、多轮对话、工具调用。你已经有了这些能力，扩展到其他场景只是换个应用层。

OpenAI 在给 Excel 和 PowerPoint 做集成，Anthropic 在做 Canvas 和 Artifacts，Google 在 Workspace 里塞 Gemini。模型厂商都在做同一件事——**把 AI 能力铺到所有知识工作场景**。作为应用层公司，你要么跟上这个节奏，要么被碾过去。

## 但这条路没那么好走

Replit 的转型看起来顺理成章，但实际执行起来有几个坑。

**第一个坑：用户心智。** 用户认为 Replit 是「写代码的地方」。你突然告诉他们「你也可以在这里做 PPT」，他们的第一反应不是「太好了」，而是「为什么？」这不是技术问题，是认知问题。就像你很难说服一个习惯用 Word 的人去 Notion 写文档——不是 Notion 不好用，是用户已经有了固定的工具选择。

**第二个坑：功能深度 vs 功能广度。** Replit 现在做了 Canvas、Apps、Sites、Slides、Videos 五个方向。每个方向都有成熟的竞品：Figma、Webflow、Gamma、Descript。如果 Replit 的 Slides 功能只是「能用」的水平，用户为什么不继续用 Gamma？除非你在某个点上做到明显更好——比如「AI 生成的 PPT 质量吊打所有竞品」，或者「代码和文档无缝联动」——否则用户没有切换的动力。

广度扩展很快，深度打磨很慢。Replit 能不能在每个方向上都做到足够好，是个问号。

第三个坑：平台 vs 工具。 Replit 想做的是平台——一个集成了所有知识工作功能的地方。但用户习惯的是工具——每个任务用最好的那个工具。平台的优势是「一站式」，劣势是「每个功能都不是最好的」。工具的优势是「专精」，劣势是「需要在多个工具之间切换」。

用户会选哪个？取决于「切换成本」和「功能差距」的对比。如果 Replit 的 Canvas 比 Figma 差 20%，但能省掉切换的麻烦，用户可能会接受。如果差 50%，用户还是会回到 Figma。

## Replit 能赢吗？

我不知道。

但我知道的是，这场仗已经开打了。不只是 Replit，所有 coding agent 公司都在往知识工作方向扩展。谁能在这条路上走得更远，取决于三件事：

- 产品整合能力 — 能不能把多个功能做成一个流畅的整体，而不是功能列表的堆砌
- 用户迁移策略 — 能不能说服用户从现有工具切换过来
- AI 能力护城河 — 当所有人都在用同样的模型，你的差异化在哪里

Replit 的优势是动作快、融资能力强、CEO 对趋势的判断准。劣势是起步晚、用户心智固化、竞争对手太多。半年估值翻三倍，说明资本看好这个方向。但估值不等于胜利。真正的考验是一年后，用户是把 Replit 当成「偶尔用用的代码工具」，还是「每天离不开的工作平台」。

## 常见问题

Q: Replit 的转型对独立开发者意味着什么？

如果你现在还在手写代码，可能需要重新思考自己的定位了。AI 已经能完成大部分编程工作，你的价值不再是「会写代码」，而是「知道该做什么」和「能判断 AI 的输出质量」。独立开发者的核心竞争力正在从「技术实现」转向「产品判断」。

Q: 其他 coding agent 公司会跟进吗？

已经在跟进了。Cursor 在测试文档协作功能，GitHub Copilot 在做项目管理集成，Codeium 在做团队协作工具。这不是 Replit 的独家策略，而是整个行业的共同方向。问题不是「会不会跟进」，而是「谁能做得更好」。

Q: 传统办公软件（Microsoft 365、Google Workspace）会被取代吗？

短期内不会。这些平台有几十年的用户积累和企业集成，不是一两年能撼动的。但长期看，如果 AI-native 的工作平台能提供明显更好的体验，用户迁移是必然的。就像当年 Slack 挑战 Email、Notion 挑战 Word 一样——不是完全取代，而是在特定场景下成为更好的选择。
