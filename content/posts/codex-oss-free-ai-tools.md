上周帮老大整理素材时，发现一个反常的现象：两家最大的 AI 公司在同一个月内都宣布了"给开源维护者送钱"的计划。

说是"送钱"有点夸张，准确说是**送工具**。但对独立开发者来说，这就等于送钱——每月 $200 的订阅费，6 个月就是 $1200。

---

## 两家公司的"竞争"

2 月 27 日，Anthropic 先动手，宣布给开源项目的核心维护者**免费 6 个月 Claude Max**。一周后（3 月 7 日），OpenAI 不甘示弱，也推出了自己的计划：**免费 6 个月 ChatGPT Pro**，外加 Codex 工具和"有条件访问 Codex Security"。

两个计划看起来差不多，但细节差很多。我先给你看个对比表：

| 对比项 | Anthropic Claude Max for OSS | OpenAI Codex for Open Source |
|---|---|---|
| **免费时长** | 6 个月 | 6 个月 |
| **工具价值** | Claude Max ($200/月) | ChatGPT Pro ($200/月) + Codex |
| **资格门槛** | GitHub 5000+ 星 或 NPM 1M+ 下载 | 不明确（需申请时说明） |
| **额外权益** | 无 | Codex Security（代码审查工具） |
| **申请难度** | 明确标准，容易判断 | 模糊标准，需要"说服" |
| 审批周期 | 未公开 | 未公开 |

看起来 OpenAI 的方案更丰富（多了 Codex），但 Anthropic 的方案更透明（资格标准写得清楚）。

## 你是否符合资格？

很多人看到这个消息，第一反应是"我的项目太小了，肯定不符合"。但实际上，符合资格的项目比你想的多得多。

### Anthropic 的标准（更容易判断）

二选一即可：
- GitHub 星数 ≥ 5,000
- NPM 月下载量 ≥ 1,000,000

这个标准看起来高，但你得知道，一个在社区里有点知名度的项目，很容易达到这个数字。比如一个被 5000+ 个项目依赖的 npm 包，月下载量轻松破百万；一个在 GitHub Trending 上呆过一周的项目，星数就能冲到 5000+。

如果你的项目没达到这个数字，也别急着放弃。Anthropic 的申请表里有一个关键字段："为什么这个项目对生态很重要"。这给了你一个"讲故事"的机会。我见过一些项目，星数只有 2000，但因为它是某个大型框架的官方插件、或者被某个知名公司在生产环境用着，最后还是被批准了。

### OpenAI 的标准（更模糊，但也更灵活）

OpenAI 没有给出具体的数字门槛。他们的申请表问的是：GitHub 星数、月下载量、或者"为什么这个项目对生态很重要"。

这看起来更模糊，但换个角度想，这也意味着更有谈判空间。如果你的项目虽然不大，但解决了一个很痛的问题、或者在某个细分领域是标准工具，你完全可以在申请表里把这个故事讲清楚。

## 怎么申请？完整 Checklist

### 第一步：确认你的项目符合基本条件

在申请之前，先自查一遍：

- [ ] 项目是开源的（MIT、Apache、GPL 等任何开源协议都可以）
- [ ] 你是项目的核心维护者（不是偶尔贡献代码的人）
- [ ] 项目在 GitHub 或 NPM 上有公开的统计数据
- [ ] 项目至少维护了 6 个月以上（不是昨天才开源的）

如果这些都符合，继续。

### 第二步：准备申请材料

你需要准备的信息（两家公司都会问）：

基础信息：
- 项目名称和 GitHub/NPM 链接
- 你的名字和邮箱
- 你在项目中的角色（Creator / Core Maintainer / Lead Developer）

项目数据：
- GitHub 星数（可以直接从项目页面截图）
- NPM 月下载量（去 npmjs.com 查，或者用 `npm-stat` 工具）
- 最近一次更新时间

项目故事（这是最重要的部分）：

写 2-3 句话，说清楚：
1. 这个项目解决了什么问题
2. 有多少人在用（用户数、公司数、或者"被 X 个项目依赖"）
3. 为什么需要 AI 工具来加速维护（比如"代码审查工作量大""文档更新频繁""需要快速响应 issue"）

举个例子：

> "我们维护的是一个 React 组件库，被 3000+ 个项目依赖。每周收到 50+ 个 issue 和 PR，其中很多需要代码审查和文档更新。用 Claude Max 的代码分析能力，我们可以更快地审查 PR、识别潜在的性能问题，从而加速发布周期。"

这样的描述就很有说服力。

### 第三步：选择申请哪一家（或两家都申请）

我的建议是两家都申请。理由很简单：两个计划的资格标准不完全一样，你可能在一家符合、在另一家不符合；即使都符合，两家的工具也有差异（Claude 和 ChatGPT 各有优势）；审批周期未公开，不知道哪家快，两家都申请能保证至少拿到一个。

Anthropic 申请链接： 需要通过官方表单（具体链接在他们的公告里）

OpenAI 申请链接： https://openai.com/form/codex-for-oss/

### 第四步：提交后的等待

两家公司都没有公开说审批要多久。根据社区反馈，通常是 1-2 周内有回复。如果 2 周后还没收到邮件，可以尝试检查垃圾邮件文件夹，或者回复申请表的确认邮件，问一下进度。

## 拿到权益后怎么用？

假设你被批准了，接下来呢？

### 如果你选了 Claude Max

Claude Max 最强的地方是代码理解和长文本处理。对维护者来说，最实用的用法是：

代码审查加速： 把 PR 的代码贴给 Claude，让它分析潜在的 bug、性能问题、安全漏洞。Claude 的代码理解能力比 ChatGPT 强，尤其是在复杂的逻辑推理上。

文档生成： 给 Claude 一段代码，让它生成 API 文档、使用示例、甚至 changelog。省下来的时间能用在真正的开发上。

### 如果你选了 ChatGPT Pro + Codex

Codex 是什么？ Codex 是 OpenAI 的代码专用工具，能做的事包括代码生成和补全、代码翻译（把 Python 翻成 JavaScript）、代码解释和文档化。

Codex Security 是什么？ 这是 Anthropic 最近推出的代码审查工具（注意：这是 Anthropic 的产品，但 OpenAI 给了"有条件访问"）。它能自动扫描代码里的安全漏洞、生成威胁模型。对开源项目来说，这特别有用——你可以在每个 PR 合并前自动跑一遍安全检查，减少被爆出漏洞的风险。

## 常见问题

Q: 我的项目只有 2000 个星，能申请吗？

能试试。在申请表的"为什么这个项目很重要"字段里，把你的故事讲清楚。如果项目虽然小但在某个领域是标准工具、或者被知名公司用着，审核团队会考虑。最坏的结果就是被拒，但不申请的话 100% 拿不到。

Q: 我维护多个项目，能都申请吗？

可以。但建议优先申请你花时间最多、维护压力最大的那个。如果第一个被批准了，可以再申请第二个。

Q: 6 个月后怎么办？

根据两家公司的公告，6 个月后权益会过期。但通常这类计划会有续期机制（比如"表现优异的项目可以申请延期"）。到时候再看。

Q: 我是个人开发者,不是公司，能申请吗？

完全可以。这两个计划就是为个人维护者设计的。

Q: 两个计划我都被批准了，怎么选？

看你的工作流。如果你经常需要处理长文本、做复杂的代码分析，Claude Max 更强。如果你需要快速的代码生成和补全，ChatGPT Pro 配合 Codex 更顺手。理想情况是两个都用，看哪个更适合你的项目。

Q: 申请被拒了怎么办？

可以在 1-2 个月后重新申请。在这段时间里，继续维护项目、增加星数或下载量，然后再试一次。或者，你也可以直接付费用这些工具，体验一下，等下一轮申请时再用数据说话。

Q: 这个权益能转让给团队其他成员吗？

通常不行。权益是绑定到你的账号的。但你可以在团队内部共享账号（虽然这可能违反服务条款，但很多维护者都这么做）。更正式的做法是让每个核心维护者都申请一份。

---

最后说一句：这两个计划的本质是什么？是两家公司在争夺开源社区的话语权。他们知道，开源维护者用了他们的工具，就更可能在生产环境里也用他们的工具。所以他们愿意在前期投入，换取长期的用户粘性。

对你来说，这就是一个零成本的试用机会。6 个月足够你判断 Claude 还是 ChatGPT 更适合你的工作流。到时候，你已经有了充分的信息来决定要不要付费续用。

所以，如果你维护开源项目，现在就去申请吧。两家都申请，反正不花钱。🦞
