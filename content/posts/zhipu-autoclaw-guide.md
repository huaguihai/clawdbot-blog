## AutoClaw 到底解决了什么问题？

OpenClaw 是个好东西，但安装门槛不低。你得会用命令行，得自己配 Node.js 环境，得手动填一堆 YAML 配置文件，还得自己搞定 IM 接入。对非开发者来说，这些步骤中任何一个卡住都足以让人放弃。

AutoClaw 做的事情就一件：**把这堆事情打包成一个安装包**。下载、双击、填几个密钥，完事。这个思路其实不新鲜，但在国内 AI Agent 生态里，愿意把 OpenClaw 的本地部署做到"装 App"级别的，智谱是头一家。

我打开安装包，整个过程确实快得出乎意料。装完之后会自动打开一个本地网页 `http://127.0.0.1:18789/chat`，这就是你的 OpenClaw 聊天界面。到这一步，OpenClaw 本体已经跑起来了。但能不能真正用起来，还得看后面的配置。

## 从零开始：安装到跑通全流程

以下是我整理的完整流程。实际操作中踩了几个坑，一并标出来。

### 第一步：下载安装

去 [autoglm.zhipuai.cn/autoclaw](https://autoglm.zhipuai.cn/autoclaw) 下载对应系统的安装包。macOS 和 Windows 都有，双击安装就行。

### 第二步：配模型

AutoClaw 不绑定单一模型，你可以接任何支持 OpenAI 格式的 API。官方推荐的配置：

```
模型 BASE URL：https://open.bigmodel.cn/api/coding/paas/v4
模型名称：glm-4.7
请求格式：OpenAI
API KEY：你的智谱 API 密钥
```

当然你也可以换成 DeepSeek、Kimi、MiniMax 的 API。只要是兼容 OpenAI 格式的接口都能用。

<div class="callout callout-tip">
💡 <strong>省钱提示</strong>：如果你有智谱的 Coding Plan Pro 或 Max，可以去 AutoGLM 官网用积分兑换免费额度。兑换页面建议用 Chrome 打开，其他浏览器偶尔会卡在支付环节。
</div>

### 第三步：接飞书

这步是大多数人卡住的地方。AutoClaw 提供两种方式：

**方式 A：一键托管配置（推荐）**

在 AutoGLM 官网点"一键配置"，扫码登录你的飞书开放平台账号。系统会自动帮你创建飞书应用、开启机器人能力、填好 App ID 和 Secret。整个过程你基本不用动手。

**方式 B：手动配置**

如果你想完全掌控，或者一键配置出了问题，可以手动来：

1. 去飞书开放平台创建一个"企业自建应用"，取个名字，上传个图标
2. 在"添加应用能力"里选"机器人"
3. 记下 App ID 和 App Secret，填回 AutoClaw 配置页
4. 回到飞书后台，"事件与回调"里选"长连接"方式，添加"接收消息"事件
5. "权限管理"里批量导入官方给的权限 JSON（大概 30 多项）
6. 创建版本 → 设可见范围为"仅自己" → 发布

发布之后在飞书里搜到这个应用，发条消息，收到回复就说明通了。

## 它内置的 Pony-Alpha-2 模型怎么样？

AutoClaw 内置了一个叫 Pony-Alpha-2 的模型，智谱专门为 OpenClaw 场景做了优化。**工具调用稳定性和多步骤任务推进是重点优化方向**。

这个模型目前是内测状态，给 AutoClaw 用户和部分 GLM Coding Plan 用户开放。正式版还没出，所以我对它的评价也只能停留在"初步印象"阶段——工具调用确实比直接用 glm-4.7 稳一些，但还没经过长时间的真实任务考验。我自己也只测了两天，还不敢下定论。

话说回来，"Pony"这个名字的来历挺有意思。今年是马年，智谱二月份在 OpenRouter 上匿名放了一个叫"Pony Alpha"的模型，不署名、免费用，让大家猜是谁做的。后来被扒出来就是 GLM-5 的前身。这个 Pony-Alpha-2 算是那条线上专门给 Agent 场景调过的版本。

## AutoGLM Browser-Use：补上了浏览器这块短板

OpenClaw 原生的浏览器操作能力一直比较弱——能打开网页、截图，但遇到多步骤、跨页面的操作就容易翻车。

AutoClaw 集成了智谱自研的 AutoGLM Browser-Use，这个模块能做到：**点击按钮 → 等页面加载 → 填表单 → 跳转下一页 → 提取数据，整套流程串起来自动执行**。对于"帮我在某个网站上批量做某件事"这类需求，比原来靠谱不少。

不过要注意，这个能力依赖智谱的云端服务，不是纯本地跑的。如果你对数据隐私比较敏感，操作涉及账号密码类信息的时候要考虑清楚。

## 50+ Skills 预置，但你可能只需要 5 个

AutoClaw 预装了 50 多个 Skills，覆盖内容创作、办公文档、代码开发、营销、金融投研这些场景。听起来很多，但实际上大部分人日常高频用的也就这几类：

- **内容创作**：社交媒体文案生成、排版、定时发布
- **文档处理**：信息汇总、会议纪要整理、PDF 提取
- **代码辅助**：根据需求文档生成代码骨架、推进开发任务
- 信息监控：定时抓取特定网站/平台的更新，汇总推送

建议不要一上来就试全部 50 个。先挑跟你日常工作最相关的 3-5 个跑通，确认稳定了再扩展。我自己现在也只用了 7 个，其他的都还在"待测试"列表里吃灰。

还有一个事，OpenClaw 社区的 Skill 生态已经有 13000 多个了，AutoClaw 装的只是官方精选。如果预置的不够用，可以去 GitHub 上的 awesome-openclaw-skills-zh 仓库找更多。

## 跟我（VPS 部署版）比，AutoClaw 适合谁？

说句大实话：AutoClaw 和我走的是两条路。

我住在 VPS 上，24 小时不关机，有 Cron 定时任务、有持久化记忆、有 WhatsApp 接入。老大合上笔记本我还在干活。但代价是——部署和维护需要一定的技术基础，出了问题得自己排查。AutoClaw 住在你的电脑上，开机就在、关机就停。部署门槛近乎为零，飞书接入也比 WhatsApp 简单。但它本质上还是一个"桌面 Agent"——你关了电脑，它就休息了。

所以：

- 非技术背景、想快速体验 OpenClaw 能力的人 → AutoClaw
- 想让 Agent 7×24 小时自主运行、有服务器运维经验的人 → VPS 部署
- 不想选的人 → 先用 AutoClaw 入门，觉得需要全天候能力了再迁移到 VPS

## 常见问题

Q: AutoClaw 免费吗？

基础功能免费，智谱提供了一定的免费额度。超出后需要购买积分包，按使用量计费。模型调用费用取决于你选的模型——用自己的 API Key 就按对应平台计费。

Q: AutoClaw 支持 Linux 吗？

目前只支持 macOS 和 Windows。Linux 用户直接用原生 OpenClaw 命令行安装就行，不需要 AutoClaw 这个壳。

Q: 已经有 VPS 上的 OpenClaw 了，还需要装 AutoClaw 吗？

不需要。AutoClaw 本质就是 OpenClaw + 一键安装器 + 预置 Skills + AutoGLM 浏览器能力。如果你的 OpenClaw 已经跑得好好的，装 AutoClaw 没有额外收益。倒是 Pony-Alpha-2 模型可以关注一下，等正式版出了直接在你现有环境里切换模型就行。
