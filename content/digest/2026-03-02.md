**1. OpenAI 公开与美国国防部合作协议细节**

来源：OpenAI Blog / TechCrunch AI

OpenAI 在官方博客发布了与美国国防部（Department of War）的合作协议全文，这是继上周五角大楼将 Anthropic 列为供应链风险后，AI 行业最受关注的事件。协议明确了三条安全红线：不用于自主武器系统、不用于大规模监控、不用于违反国际人道法的行动。同时披露了 AI 系统在机密环境中的部署方式和法律保护条款。

这份协议的公开是对 Anthropic 拒绝军方合作立场的直接回应。CEO Sam Altman 在接受采访时表示"技术中立，关键在于使用规则"。协议发布后，Claude 在 App Store 排名从第 2 位升至第 1 位，显示出部分用户用"用脚投票"支持 Anthropic 的立场。对独立开发者而言，这场争议揭示了一个现实：当你的 AI 产品做大后，"为谁服务"将成为无法回避的选择题。

关键数据：OpenAI 与国防部签约、Anthropic 被列为供应链风险、Claude 升至 App Store 第 1 位

🔗 [OpenAI 协议原文](https://openai.com/index/our-agreement-with-the-department-of-war) | [TechCrunch 报道](https://techcrunch.com/2026/03/01/openai-shares-more-details-about-its-agreement-with-the-pentagon/)

**2. Anthropic 发布 Claude Code：终端里的 AI 编程助手**

来源：GitHub Trending

Anthropic 开源了 Claude Code，一个运行在终端中的 agentic 编码工具。它能理解代码库上下文、执行常规任务、解释复杂代码、处理 git 工作流。项目用 Shell 编写，今日获得 699 个 Star。

这是 Anthropic 在"五角大楼事件"后的第一个重要产品发布。与 GitHub Copilot 和 Cursor 不同，Claude Code 不是 IDE 插件，而是独立的命令行工具——这意味着它可以跨编辑器使用，也能集成到 CI/CD 流程中。对于习惯终端操作的开发者，这是一个值得尝试的选择。配合昨天报道的 Ruflo 和 Deer-Flow 多智能体框架，终端正在成为 AI 编程的新战场。

关键数据：699 Star/天、Shell 编写、支持 git 工作流

🔗 [GitHub 仓库](https://github.com/anthropics/claude-code)

**3. LLM 应用开发资源库登顶 GitHub Trending**

来源：GitHub Trending

awesome-llm-apps 是一个收集 LLM 应用开发案例的 Python 项目，涵盖 AI Agent、RAG、多模态应用等场景，支持 OpenAI、Anthropic、Gemini 和开源模型。今日获得 635 个 Star。

这个项目的价值在于"可直接运行的代码示例"——不是理论教程，而是真实的应用模板。对于想快速上手 LLM 开发的独立开发者，这是一个比文档更实用的学习资源。项目按功能分类（Agent、RAG、多模态），每个示例都有完整代码和依赖说明，可以直接 fork 后修改成自己的产品原型。

关键数据：635 Star/天、支持 4 类主流 LLM、包含 Agent 和 RAG 实战案例

🔗 [GitHub 仓库](https://github.com/Shubhamsaboo/awesome-llm-apps)
