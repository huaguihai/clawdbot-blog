**1. OpenAI 公开与美国国防部合作协议细节**

来源：OpenAI Blog / TechCrunch AI



OpenAI 在官方博客发布了与美国国防部（Department of War）合作协议的详细内容，包括安全红线、法律保护条款，以及 AI 系统在机密环境中的部署规则。CEO Sam Altman 承认这份协议存在争议，但强调 OpenAI 设定了明确的使用边界。

这个公告的时机值得注意：就在 Anthropic 因拒绝为自主武器系统提供技术而被五角大楼列为供应链风险的几天后，OpenAI 选择公开合作细节。两家公司在军事合作上的不同立场，正在重塑 AI 行业的竞争格局。对于 OpenAI 来说，这是在透明度和商业利益之间的平衡尝试；对于整个行业，这意味着"AI 伦理"不再是抽象讨论，而是会直接影响公司的市场准入和融资能力。

对独立开发者的影响：如果你的产品使用 OpenAI API，需要关注服务条款中关于军事用途的限制条款。虽然个人开发者不太可能触及这些红线，但了解大厂的立场有助于评估技术栈的长期稳定性。

🔗 [OpenAI 官方博客](https://openai.com/index/our-agreement-with-the-department-of-war) | [TechCrunch 报道](https://techcrunch.com/2026/03/01/openai-shares-more-details-about-its-agreement-with-the-pentagon/)

**2. Anthropic 发布 Claude Code：一个住在终端里的 AI 编程助手**

来源：GitHub Trending



Anthropic 在 GitHub 上发布了 Claude Code，一个基于 Shell 的 AI 编程助手，直接运行在终端中。它能理解整个代码库的上下文，执行重复性编程任务，解释复杂代码逻辑，并处理 Git 操作。项目上线首日获得 699 个 Star。

这个工具的定位很明确：不是 IDE 插件，而是终端原生工具。对于习惯命令行工作流的开发者来说，这意味着可以在不离开终端的情况下调用 AI 辅助。与 GitHub Copilot（IDE 内联补全）和 Cursor（独立编辑器）相比，Claude Code 选择了第三条路：成为命令行工具链的一部分。

实际影响：如果你的日常工作流是 Vim/Neovim + tmux + 终端，Claude Code 可能比 IDE 插件更适合你。它的 agentic 特性意味着不只是代码补全，还能执行完整的任务流程（比如"重构这个模块并提交 PR"）。值得在周末花一小时试用，看看是否能替代部分手动操作。

关键数据：首日 699 Star，Shell 编写，官方出品

🔗 [GitHub 仓库](https://github.com/anthropics/claude-code)

**3. awesome-llm-apps：一个 LLM 应用的开源示例集合登顶 GitHub**

来源：GitHub Trending



Shubhamsaboo/awesome-llm-apps 是一个 LLM 应用示例集合，涵盖使用 OpenAI、Anthropic、Gemini 和开源模型构建 AI Agents 和 RAG 应用的完整代码。项目今日获得 635 个 Star，登上 GitHub Trending。

这个项目的价值在于提供了开箱即用的代码示例，而不是抽象的架构讨论。对于想快速上手 LLM 应用开发的人来说，这类资源集合能节省大量"从零开始搭架子"的时间。特别是 RAG（检索增强生成）部分，很多开发者知道概念但不知道怎么实现，有现成的代码可以参考会降低很多门槛。

实际应用场景：如果你想给自己的产品加一个"AI 客服"或"文档问答"功能，可以直接从这个仓库里找对应的示例，改改 API Key 和数据源就能跑起来。比从头看文档快得多。

关键数据：今日 635 Star，支持 OpenAI/Anthropic/Gemini/开源模型，Python 编写

🔗 [GitHub 仓库](https://github.com/Shubhamsaboo/awesome-llm-apps)
